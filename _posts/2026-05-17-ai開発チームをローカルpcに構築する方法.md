# AI開発チームをローカルPCに構築する方法

## 1. 導入

近年、AIを活用した開発支援ツールが多数登場していますが、多くのサービスはクラウド依存であり、オフライン環境やプライベートなコードベースでの利用に制約があります。また、チームでAIエージェントを活用する際、各メンバーのローカル環境で統一した設定を共有できると、効率的な開発フローを実現できます。

本記事では、Node.jsベースの軽量なフレームワークを用いて、ローカルPC上に複数のAIエージェントが協調して動作する「開発チーム」を構築する方法を解説します。これにより、コードレビュー、テスト生成、リファクタリング提案などを自動化する基盤が手に入ります。

## 2. 前提条件

以下の環境が整っていることを確認してください。

- **OS**: macOS / Linux / Windows (WSL2推奨)
- **Node.js**: v18.0.0以上（LTS推奨）
- **パッケージマネージャ**: npm または yarn
- **OpenAI APIキー**: 有効なAPIキー（ローカルLLMを使う場合は不要）
- **TypeScript**: 基本的な文法理解

使用する主なライブラリは以下の通りです。

```json
{
  "dependencies": {
    "openai": "^4.20.0",
    "commander": "^11.0.0",
    "chalk": "^5.3.0"
  },
  "devDependencies": {
    "typescript": "^5.3.0",
    "@types/node": "^20.0.0"
  }
}
```

## 3. 実装手順

### ステップ1: プロジェクトの初期化

```bash
mkdir ai-dev-team
cd ai-dev-team
npm init -y
npm install openai commander chalk
npm install -D typescript @types/node
npx tsc --init
```

### ステップ2: TypeScriptの設定

`tsconfig.json` の主要設定を確認・修正します。

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "node",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true
  }
}
```

### ステップ3: エージェント管理クラスの作成

`src/agents.ts` を作成し、複数のAIエージェントを管理する基底クラスを実装します。

## 4. コードサンプル

### サンプル1: 基本エージェントクラス

```typescript
// src/agents.ts
import OpenAI from 'openai';

interface AgentConfig {
  name: string;
  role: string;
  model?: string;
}

export class AIAgent {
  private client: OpenAI;
  public name: string;
  public role: string;
  private model: string;

  constructor(config: AgentConfig) {
    this.client = new OpenAI({
      apiKey: process.env.OPENAI_API_KEY,
    });
    this.name = config.name;
    this.role = config.role;
    this.model = config.model || 'gpt-4o-mini';
  }

  async execute(task: string, context?: string): Promise<string> {
    const systemPrompt = `あなたは ${this.role} として行動するAIエージェントです。
    名前は ${this.name} です。与えられたタスクを正確に実行してください。`;

    const messages: OpenAI.Chat.Completions.ChatCompletionMessageParam[] = [
      { role: 'system', content: systemPrompt },
    ];

    if (context) {
      messages.push({ role: 'user', content: `コンテキスト: ${context}` });
    }
    messages.push({ role: 'user', content: task });

    const response = await this.client.chat.completions.create({
      model: this.model,
      messages: messages,
      temperature: 0.3,
    });

    return response.choices[0]?.message?.content || '';
  }
}
```

### サンプル2: コードレビューエージェント

```typescript
// src/reviewer.ts
import { AIAgent } from './agents.js';

export class CodeReviewerAgent extends AIAgent {
  constructor() {
    super({
      name: 'reviewer',
      role: 'シニアソフトウェアエンジニア。コードレビューと改善提案の専門家',
      model: 'gpt-4o',
    });
  }

  async reviewCode(code: string): Promise<string> {
    const task = `以下のコードをレビューしてください。
    以下の観点で評価し、改善点があれば具体的な修正案を示してください。
    - バグの可能性
    - パフォーマンス問題
    - セキュリティリスク
    - コードスタイル違反
    - テスト容易性

    コード:
    \`\`\`typescript
    ${code}
    \`\`\``;

    return await this.execute(task);
  }
}
```

### サンプル3: メイン実行ファイル

```typescript
// src/index.ts
import { Command } from 'commander';
import { CodeReviewerAgent } from './reviewer.js';
import fs from 'fs/promises';
import chalk from 'chalk';

const program = new Command();

program
  .name('ai-dev-team')
  .description('ローカルAI開発チーム')
  .version('1.0.0');

program
  .command('review <filepath>')
  .description('指定されたファイルのコードレビューを実行')
  .action(async (filepath: string) => {
    try {
      const code = await fs.readFile(filepath, 'utf-8');
      const reviewer = new CodeReviewerAgent();
      
      console.log(chalk.blue(`\n🔍 ${filepath} のコードレビューを開始します...\n`));
      
      const reviewResult = await reviewer.reviewCode(code);
      
      console.log(chalk.green('=== レビュー結果 ==='));
      console.log(reviewResult);
    } catch (error) {
      console.error(chalk.red('エラーが発生しました:'), error);
    }
  });

program.parse(process.argv);
```

## 5. ハマりやすいポイント

### APIキーの管理
環境変数 `OPENAI_API_KEY` を設定しないとエラーになります。`.env` ファイルを使うか、シェルの設定ファイルに追加してください。

```bash
export OPENAI_API_KEY="sk-xxxxx"
```

### TypeScriptの実行方法
ES Modulesを使用する場合、`package.json` に以下を追加する必要があります。

```json
{
  "type": "module"
}
```

また、`tsc` でコンパイル後、Node.jsで実行する際は拡張子 `.js` が必要です（上記コードでは `.js` でインポートしています）。

### レート制限
無料枠のAPIキーでは、1分あたりのリクエスト数に制限があります。大量のコードを一度にレビューする場合は、適宜遅延を入れるか、バッチ処理を検討してください。

### コンテキスト長の制限
GPT-4o-miniは比較的長いコンテキストを扱えますが、巨大なファイル（数千行）を一度に送信するとトークン制限に達する可能性があります。ファイルを分割して処理するロジックを追加することを推奨します。

## 6. まとめ

ローカルPC上にAI開発チームを構築することで、以下のメリットが得られます。

- コードレビューの自動化による品質向上
- オフライン環境でも動作可能（ローカルLLM利用時）
- チーム全体で統一されたレビュー基準の適用
- プライベートコードの外部送信を最小化

今回紹介した基本的なフレームワークを拡張すれば、テスト自動生成、リファクタリング提案、ドキュメント生成など、さまざまなタスクをAIエージェントに任せることができます。実際の開発フローに組み込む際は、段階的に導入し、人間のレビューと併用することをおすすめします。
