# Bunで始めるAIエージェント開発：5分で動くサンプル

## 1. 導入

近年、AIエージェント（自律的にタスクを実行するAIシステム）への関心が高まっています。しかし、実際に開発しようとすると、環境構築やAPI連携の複雑さに悩まされることが少なくありません。

そこで本記事では、高速なJavaScript/TypeScriptランタイム「Bun」を使って、最小限のコードでAIエージェントを動かす方法を紹介します。BunはNode.js互換でありながら起動が高速で、パッケージ管理やテストランナーも内蔵しているため、プロトタイピングに最適です。

具体的には、以下の2つのサンプルを作成します。

1. **シンプルなチャットエージェント**: ユーザーの質問に対してAIが回答を生成する
2. **タスク実行エージェント**: 与えられた指示を解釈し、ファイル操作などのアクションを実行する

どちらも5分あれば動作確認が可能です。

## 2. 前提条件

- **Bun**がインストールされていること（`bun --version` で確認）
  - 未インストールの場合は公式サイト（https://bun.sh）からインストールしてください
- **OpenAI APIキー**を取得していること（環境変数 `OPENAI_API_KEY` に設定）
- TypeScriptの基本的な知識

## 3. 実装手順（ステップバイステップ）

### 3-1. プロジェクトの作成

任意のディレクトリで以下のコマンドを実行します。

```bash
bun init -y
```

これで `package.json` と `index.ts` が生成されます。

### 3-2. 必要なライブラリのインストール

OpenAIの公式SDKをインストールします。

```bash
bun add openai
```

### 3-3. 環境変数の設定

`.env` ファイルを作成し、APIキーを記述します。

```bash
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Bunはデフォルトで `.env` ファイルを読み込むため、追加の設定は不要です。

## 4. コードサンプル

### サンプル1: シンプルなチャットエージェント

`index.ts` に以下のコードを記述します。

```typescript
import OpenAI from "openai";

const openai = new OpenAI();

async function chatAgent(prompt: string): Promise<string> {
  const response = await openai.chat.completions.create({
    model: "gpt-4o-mini",
    messages: [
      {
        role: "system",
        content: "あなたは親切なアシスタントです。簡潔に回答してください。",
      },
      { role: "user", content: prompt },
    ],
    temperature: 0.7,
    max_tokens: 200,
  });

  return response.choices[0]?.message?.content ?? "応答を取得できませんでした。";
}

// 実行例
const userPrompt = "TypeScriptで非同期処理を書くときの注意点を教えてください。";
chatAgent(userPrompt).then((answer) => {
  console.log("AI:", answer);
});
```

**実行方法**: `bun run index.ts`

このコードでは、OpenAIのChat Completion APIを使ってユーザーの質問に回答します。`gpt-4o-mini` は低コストで高速なモデルなので、プロトタイプに適しています。

### サンプル2: タスク実行エージェント

次に、AIが指示を解釈して実際にファイル操作を行うエージェントを作成します。`agent.ts` を作成してください。

```typescript
import OpenAI from "openai";
import { writeFile, readFile } from "node:fs/promises";

const openai = new OpenAI();

// 利用可能なツールの定義
const tools = [
  {
    type: "function" as const,
    function: {
      name: "create_file",
      description: "指定された内容でファイルを作成します",
      parameters: {
        type: "object",
        properties: {
          filename: { type: "string", description: "作成するファイル名" },
          content: { type: "string", description: "ファイルに書き込む内容" },
        },
        required: ["filename", "content"],
      },
    },
  },
  {
    type: "function" as const,
    function: {
      name: "read_file",
      description: "指定されたファイルの内容を読み取ります",
      parameters: {
        type: "object",
        properties: {
          filename: { type: "string", description: "読み取るファイル名" },
        },
        required: ["filename"],
      },
    },
  },
];

// ツールの実際の処理
async function executeTool(name: string, args: Record<string, unknown>): Promise<string> {
  switch (name) {
    case "create_file": {
      const { filename, content } = args as { filename: string; content: string };
      await writeFile(filename, content, "utf-8");
      return `ファイル ${filename} を作成しました。`;
    }
    case "read_file": {
      const { filename } = args as { filename: string };
      const content = await readFile(filename, "utf-8");
      return `ファイル ${filename} の内容:\n${content}`;
    }
    default:
      throw new Error(`未知のツール: ${name}`);
  }
}

async function taskAgent(task: string): Promise<void> {
  // 1. AIにタスクを解釈させる
  const response = await openai.chat.completions.create({
    model: "gpt-4o-mini",
    messages: [
      {
        role: "system",
        content: "あなたはファイル操作ができるアシスタントです。与えられたタスクを達成するために、適切なツールを選択・実行してください。",
      },
      { role: "user", content: task },
    ],
    tools,
    tool_choice: "auto",
  });

  const message = response.choices[0]?.message;

  // 2. ツール呼び出しがあれば実行
  if (message?.tool_calls) {
    for (const call of message.tool_calls) {
      const args = JSON.parse(call.function.arguments);
      const result = await executeTool(call.function.name, args);
      console.log(`ツール実行結果: ${result}`);
    }
  } else {
    console.log("AIからの応答:", message?.content);
  }
}

// 実行例: タスクを指定
taskAgent("hello.txtというファイルを作成し、中に「Hello, AI Agent!」と書き込んでください。");
```

**実行方法**: `bun run agent.ts`

このサンプルでは、AIが「ファイルを作成する」「ファイルを読み取る」という2つのツールを選択・実行できるようになっています。`tool_choice: "auto"` により、AIがタスクに応じて適切なツールを判断します。

## 5. ハマりやすいポイント

### 5-1. APIキーの設定漏れ

環境変数 `OPENAI_API_KEY` が設定されていないと、`401 AuthenticationError` が発生します。`.env` ファイルが正しく読み込まれているか確認してください。Bunでは `bun run` 時に自動で読み込まれますが、念のためコード内で `console.log(process.env.OPENAI_API_KEY)` を実行して確認するのも手です。

### 5-2. ツールの引数の型

サンプル2で `executeTool` 関数に渡す引数は、AIが生成したJSONをパースしたものです。プロパティ名のスペルミスや型の不一致があるとエラーになります。OpenAIのレスポンスを `console.log` で確認しながら開発するとスムーズです。

### 5-3. BunのファイルシステムAPI

BunはNode.jsの `fs/promises` をそのまま使えますが、`bun:fs` という独自のAPIも提供しています。互換性を重視するならNode.js標準のAPIを使うことをおすすめします。

### 5-4. モデルの選択

`gpt-4o-mini` は低コストですが、複雑なタスクでは `gpt-4o` の方が安定します。開発初期は `gpt-4o-mini` で試し、必要に応じてアップグレードすると良いでしょう。

## 6. まとめ

本記事では、Bunを使ってAIエージェントをわずか5分で動かす方法を紹介しました。

- **サンプル1**では、OpenAI APIを使ってシンプルなチャットエージェントを実装しました
- **サンプル2**では、AIがツールを選択・実行するタスク実行エージェントを実装しました
